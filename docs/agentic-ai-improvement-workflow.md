# Agentic AI Improvement Workflow

## Purpose

This document defines how `agile_v_skills` should be evaluated against trending agentic AI repositories and patterns.

## Daily scan inputs

- Trending GitHub repositories related to agentic AI
- Emerging skill, tool-use, orchestration, evaluation, memory, and planning patterns
- Changes relevant to Agile-V skills and developer workflows

## Evaluation criteria

Each candidate improvement should be assessed for:

1. Relevance to Agile-V skill execution
2. Expected user impact
3. Implementation complexity
4. Testability
5. Documentation requirements
6. Compatibility with existing skill structure

## Required daily output

For each actionable improvement, provide:

- Source repository or pattern
- Why it matters for `agile_v_skills`
- Concrete change proposal
- Files or components likely affected
- Expected benefit
- Risks or migration concerns
- Suggested branch name
- Suggested commit plan

## PR readiness checklist

- Improvement is specific and traceable to a source pattern
- Scope is small enough for review
- Documentation or examples are updated when behavior changes
- Tests or validation steps are included when applicable
- The PR explains why the change improves Agile-V skill quality
