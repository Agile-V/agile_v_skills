# Agile V™ Skills Library v1.6: Runtime Governance Contracts

**Release Date:** April 26, 2026  
**Version:** 1.6  
**License:** CC-BY-SA-4.0

## Summary

Version 1.6 adds standardized **runtime governance contracts** and updates core docs/routing so teams can apply them consistently across projects.

The release formalizes machine-readable structures for trace logs, eval gating, policy-as-code, and durable Human Gate checkpoints. It also introduces ready-to-copy templates for immediate use in `.agile-v/`.

## What's New

### 1. Runtime schema specification (`docs/agile-v-runtime/01_SCHEMAS.md`)

New normative schema document for runtime artifacts under `.agile-v/`.

**Covers:**
- `TRACE_LOG.md` append-only span format
- `EVAL_RESULTS.md` header + run rows + gate constraints
- `POLICY.yaml` policy-as-code contract
- `CHECKPOINTS.md` durable HITL interrupt lifecycle
- `FT-*` failure taxonomy mappings

### 2. Ready-to-copy templates (`templates/agile-v/`)

New templates to bootstrap runtime contracts in project repos:
- `POLICY.example.yaml`
- `TRACE_LOG.example.md`
- `EVAL_RESULTS.example.md`
- `CHECKPOINTS.example.md`
- `README.md` mapping examples to `.agile-v/` target filenames

### 3. Skill contract updates (v1.4 skill metadata)

Selected core skills were updated to enforce runtime governance behavior:
- `agile-v-core`: Eval Gate rule, policy+trace rule, durable HITL checkpoint rule
- `red-team-verifier`: FT-code requirement, EvalGate block, `EVAL_RESULTS.md` handling
- `compliance-auditor`: policy/eval/checkpoint evidence checks at Gate 2
- `agile-v-compliance`: checkpoint SLA + resume-token linkage requirements
- `agile-v-pipeline`: Gate 2 prerequisites and gate pause/resume handoff references

### 4. Routing and docs updates

- `SKILL_ROUTING_GUIDE.md` now includes:
  - Eval gate evidence intent routing
  - Policy/checkpoint interruption intent routing
- `docs/README.md` adds runtime schema navigation (`RUN-001`)

## Recommended Adoption

For existing projects:
1. Copy templates from `templates/agile-v/` into project `.agile-v/`.
2. Set `policy_version` and thresholds for your environment.
3. Require `eval_gate_status` PASS or WAIVED evidence before Gate 2 approval.
4. Enforce checkpoint `resume_token` pairing with `APPROVALS.md`.

For new projects:
1. Initialize `.agile-v/` from templates at project kickoff.
2. Treat `TRACE_LOG.md` and `EVAL_RESULTS.md` as append-only evidence streams.
3. Make FT-code tagging mandatory on `VER-*` lines.

## Breaking Changes

**None.** v1.6 is additive for repository docs/templates and compatible with existing workflows.

## Deprecations

**None.**

## Known Limitations

1. Runtime contracts are provided as templates/specs; each team must set policy values and thresholds.
2. Existing projects may need one setup cycle to backfill policy/eval/checkpoint artifacts.
3. Automated enforcement depends on how strictly downstream agents and CI gates validate these files.
